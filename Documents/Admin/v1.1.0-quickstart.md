# FluentMetric AI v1.1.0 — Install & Quick Start

Salesforce-native dashboard for Einstein Generative AI Audit & Feedback data, with Wallet-based actuals and (new in v1.1.0) a self-service rate-card upload flow for non-Wallet orgs.

> **Install URL:** https://login.salesforce.com/packaging/installPackage.apexp?p0=04tHn000001NtjYIAS
> For sandboxes, swap `login` for `test` in the URL.
>
> **Status:** v1.1.0 ships as a **beta** unlocked package. Salesforce restricts beta packages to **sandbox and Developer Edition orgs only** — production orgs cannot install via the URL above. To install into a production org, either (a) wait for v1.1.1 (test-debt fix in flight) which will promote to a non-beta release, or (b) deploy via `sf project deploy start --source-dir force-app --target-org <prod-alias>` directly from the repo. The v1.1.0 rate-card tests themselves are clean (49/49 passing); the promotion blocker is unrelated test debt in `AiInsightsServiceTest`.

---

## What's in v1.1.0

- **Lightning App** — *FluentMetric AI* with 5 tabs (Overview, Adoption, Explorer, Safety, Cost)
- **19 LWCs** — usage, prompts, tokens, content safety, cost analysis, drill panels
- **Wallet-first cost engine** — actuals from `TenantEnrichedUsageEvent__dll` when Digital Wallet is enabled, otherwise tier-based estimates
- **NEW: Rate-card upload flow** — admins upload the official Salesforce Flex Credits Rate Card PDF, an LLM extracts the 8 multipliers, and the cost-settings update in one click. Non-Wallet orgs no longer need to hand-edit fields in Setup
- **New audit object** — `RateCardUpload__c` retains every uploaded PDF, parsed JSON, per-field diff, and a one-click revert

## Before you install

### 1. Org requirements

- **Salesforce edition:** Enterprise, Unlimited, or Developer
- **Data Cloud** provisioned (the Einstein Audit & Feedback DMOs live there)
- **API version 62.0+** (orgs at 66 are fine — we ship at 62)

### 2. Optional but recommended

- **Salesforce Digital Wallet** + Consumption Tagging app (production orgs only) — unlocks ACTUAL cost figures instead of estimates
- **Einstein Platform** (or Agentforce) license — unlocks the rate-card upload flow

If neither of the above is available, the app still installs and runs — you'll just see tier-based estimates without the upload-to-refresh shortcut.

## Step 1 — Install the package

1. Click the install URL above while logged in as a System Admin
2. Choose **Install for Admins Only**
3. When prompted about third-party access, approve
4. Wait ~3–5 minutes — installation status shows in **Setup → Installed Packages**

You'll get an email when the install completes.

---

## Step 2 — Assign the permission set

Every user who needs the dashboards (or wants to upload rate cards) needs `FluentMetric AI User`:

```
Setup → Permission Sets → FluentMetric AI User → Manage Assignments → Add Assignments
```

Pick the users, click **Assign**.

The permission set grants:

- The `FluentMetric AI` Lightning App
- All 19 LWCs and supporting Apex classes
- Read on the Einstein Audit & Feedback DMOs
- Read/Edit on `FluentMetric_Cost_Settings__c`
- Full CRU on `RateCardUpload__c`

---

## Step 3 — Open the app

1. App Launcher → search **FluentMetric AI** → click
2. You'll land on the **Overview** tab with 5 sub-tabs across the top

If you see *"Cost metrics are off"* on the Cost tab, that's expected — opt-in is in step 4.

---

## Step 4 — Turn on cost metrics (admin opt-in)

Cost figures default to **off** so first-load doesn't imply a measurement that hasn't been opted into.

```
Setup → Custom Settings → FluentMetric Cost Settings → Manage → New
```

Set:

| Field | Value |
|---|---|
| `Enable Cost Metrics` | ✅ on |
| `Flex Credit USD Rate` | `0.004` (default Salesforce rate; adjust if your contract differs) |
| `Discount Percent` | `0` (or your contracted discount, e.g. `25`) |
| `Default Fallback Model` | `Standard-Default` |
| `Enable Wallet Costs` | ✅ on (only if Digital Wallet is enabled in your org) |

Save. Refresh the Cost tab — tiles now render figures.

---

## Step 5 — (Wallet orgs only) Verify ACTUAL figures

If `Enable Wallet Costs` is on and your org has the Consumption Tagging app installed, the Cost tab shows a green callout:

> ✅ **Actuals from Salesforce Digital Wallet.** These Flex Credits are read directly from `TenantEnrichedUsageEvent` — the same data Salesforce uses to bill your org.

Tiles get an **ACTUAL** badge instead of `ESTIMATED` / `FALLBACK`. The rate-card upload button is hidden because Wallet figures are billing-grade.

## Step 6 — (Non-Wallet orgs) Wire up the rate-card upload flow

This is the v1.1.0 headline feature. Skip if you're on Wallet.

### 6a. Confirm Einstein is on

Run this in **Developer Console → Debug → Open Execute Anonymous Window**:

```apex
ConnectApi.EinsteinLLMGenerationsInput input = new ConnectApi.EinsteinLLMGenerationsInput();
input.messages = new List<ConnectApi.EinsteinMessage>{ new ConnectApi.EinsteinMessage() };
input.messages[0].role = 'user';
input.messages[0].content = 'Say hello.';
ConnectApi.EinsteinLLMGenerations out =
    ConnectApi.EinsteinLLM.generateMessages('sfdc_ai__DefaultGPT4Omni', input);
System.debug('LLM said: ' + out.generations[0].text);
```

**Expected:** debug log shows `LLM said: Hello…`.

If it throws, work through:

| Error | Fix |
|---|---|
| `NoAccessException` | Assign perm set **Einstein Generative AI User** to your user |
| `Method does not exist… generateMessages` | Setup → Einstein Setup → enable **Einstein Generative AI** |
| `Einstein is not turned on` | Setup → Einstein Setup → toggle the master switch |
| `No active model found` | Setup → Einstein Setup → Models → activate one model (note its API name for step 6c) |

You may also need the **Einstein GPT Platform** Permission Set License assigned by your AE.

### 6b. Confirm the active model name

```
Setup → Einstein Setup → Models
```

Note the **API Name** of an Active model — typical values: `sfdc_ai__DefaultGPT4Omni`, `sfdc_ai__DefaultOpenAIGPT4Omni`, `sfdc_ai__DefaultClaudeSonnet`.

### 6c. Wire the LLM call into the package

Open `RateCardImportService` in your org:

```
Setup → Apex Classes → RateCardImportService → Edit
```

Find the `callEinstein` method (inside the `EinsteinLlmAdapter` inner class). Replace its body with:

```apex
private String callEinstein(String systemPrompt, String userPrompt) {
    ConnectApi.EinsteinLLMGenerationsInput input = new ConnectApi.EinsteinLLMGenerationsInput();
    ConnectApi.EinsteinMessage sysMsg = new ConnectApi.EinsteinMessage();
    sysMsg.role = 'system';
    sysMsg.content = systemPrompt;
    ConnectApi.EinsteinMessage userMsg = new ConnectApi.EinsteinMessage();
    userMsg.role = 'user';
    userMsg.content = userPrompt;
    input.messages = new List<ConnectApi.EinsteinMessage>{ sysMsg, userMsg };

    ConnectApi.EinsteinLLMGenerations out =
        ConnectApi.EinsteinLLM.generateMessages('sfdc_ai__DefaultGPT4Omni', input);
    return out.generations[0].text;
}
```

Replace `sfdc_ai__DefaultGPT4Omni` with the API name from 6b. Save.

### 6d. Test the upload flow

1. Cost tab → click **Upload latest rate card** in the yellow estimates banner (or **Refresh from rate card** below the Estimated USD tile)
2. Drop in the latest official `Flex_Credits_Rate_Card.pdf` from Salesforce Help
3. Review the diff — you should see all 8 multipliers (`standard_action`, `custom_action`, `standard_voice_action`, `custom_voice_action`, `starter_prompt`, `basic_prompt`, `standard_prompt`, `advanced_prompt`)
4. Add a note (e.g. *"Q4 2025 refresh"*), click **Apply**
5. Toast: *"Rate card applied."* — tiles re-render with new math

The PDF, parsed JSON, and per-field diff are now in `RateCardUpload__c`. To roll back, query the record and call `RateCardImportService.revert(id)` from anonymous Apex (a UI for this is on the v1.2 roadmap).

## Tab-by-tab quick start

### Overview
Headline KPIs across all selected dates: requests, unique users, total tokens, cost (when enabled). Use the **date-range filter** in the top right — every tab listens.

### Adoption
Day-over-day usage. Active users, new users, top prompts, prompt-output volume. Drill into any user or prompt name to see their full history.

### Explorer
Pivot freely. Pick a dimension (User / Prompt / Model / Day) plus a measure (Requests / Tokens / Cost) and a top-N. The query is built dynamically from the filter rail.

### Safety
Content safety scores from Einstein's classifier. Trends, top flagged prompts, prompts breaching threshold. Click any row to see the original request.

### Cost
USD + Flex Credits with a confidence badge:

| Badge | Meaning |
|---|---|
| **ACTUAL** | Read from Wallet — billing-grade |
| **HIGH** | Exact rate-card match |
| **ESTIMATED** | Prefix match (e.g. `gpt-4o-2024-11-20` → `gpt-4o`) |
| **FALLBACK** | No match — using `Default Fallback Model` |
| **NOT_COSTED** | Filter excluded all known models |

If you see persistent `FALLBACK` / `ESTIMATED`, that's the signal to upload a fresh rate card (step 6d).

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| Tabs are blank | DMOs are empty | Generate some Einstein traffic — the data only flows after a request hits Audit & Feedback |
| *"Cost metrics are off"* | Custom setting default | Step 4 |
| Cost tiles show `$0.00` everywhere | All requests matched unmetered models | Expected — internal Salesforce classifiers cost $0 |
| Upload throws *"Could not parse the uploaded rate card"* | Einstein not wired (step 6c) or Einstein call failed | Re-run 6a anonymous Apex; check debug logs |
| Upload throws *"Empty response from rate-card parser"* | LLM returned non-JSON | Check the Prompt Template's strict-JSON system prompt is intact — the model occasionally hallucinates a code fence |
| Permission set won't assign | User lacks Salesforce license type | `FluentMetric AI User` requires standard Salesforce license |
| `RateCardUpload__c` records not visible in app | No tab — admin browsing only | Setup → Object Manager → RateCardUpload — view records there for now |

## Upgrade notes (1.0.x → 1.1.0)

- **Schema:** 4 new fields on `FluentMetric_Cost_Settings__c` (`Custom_Action_FC__c`, `Standard_Voice_Action_FC__c`, `Custom_Voice_Action_FC__c`, `Tier_Starter_FC__c`). New fields default to the current Salesforce rate-card values; no action needed.
- **New object:** `RateCardUpload__c` — empty until you upload your first rate card.
- **Permission set:** `FluentMetric AI User` is updated automatically — already-assigned users pick up the new perms on next page load.
- **Custom setting values:** preserved through upgrade.
- **No data migration** required.
